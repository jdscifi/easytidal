Credential Setup

To configure encrypted credentials for Tidal API access:

$ python setup_credentials.py

This will securely encrypt your username and password and store them in environment variables.

API Configuration

Update the API configuration in the .env file:

Environment Variables (.env file):
  • API_BASE_URL - Your Tidal Automation server URL
  • JOB_DIRECTORY - The job directory to monitor
  • CACHE_EXPIRY_HOURS - Cache expiration time (default: 24)
# Example .env file content:
API_BASE_URL=https://your-tidal-server.com
JOB_DIRECTORY=/production/etl
CACHE_EXPIRY_HOURS=24
Test Configuration

After setting up credentials and configuration, test the connection:

$ python test_credentials.py

This will verify that the credentials are correctly configured and the API is accessible.

Troubleshooting

  • Verify the API base URL is correct and accessible
  • Check network connectivity to the Tidal server
  • Ensure firewalls allow outbound connections
  • Verify SSL certificates if using HTTPS

  • Verify username and password are correct
  • Check if account has API access permissions
  • Ensure account is not locked or expired
  • Verify the job directory path exists and is accessible

  • Check if Tidal API is enabled and running
  • Verify API endpoints are accessible
  • Ensure API version compatibility
  • Check Tidal server logs for errors
